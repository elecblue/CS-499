Code Review & Project Proposal

A description and screencast walkthrough of my original CS-340 submission, the Grazioso Salvare animal rescue dashboard.

The original artifact for my capstone was sourced from CS-340, Client/Server Development, where the final project involved building a data-driven dashboard for Grazioso Salvare, a fictional company that trains animals for rescue operations. The application, developed using Python and the Dash web framework, provided three main functions: displaying "animal outcome" data in a dynamic table that can be filtered by "Rescue Type" (such as "Water Rescue," "Disaster Response," etc.), visualizing breed distributions by rescue type via a pie chart, and a map that displays a selected animal's rescue location.

Project Proposal / Enhancement Plan

My proposal for this capstone project centered on rebuilding the original Dash application using a contemporary tech stackâ€”specifically SvelteKit running on Bun using TypeScript throughout. The goal was to enhance performance, maintainability, and the user experience while preserving the core functionalities of the original application. Key enhancements included implementing a more responsive and interactive UI, optimizing data handling with Drizzle ORM, and ensuring secure coding practices throughout the application.

A Dash-powered web application running from a Jupyter notebook was, in my opinion, a questionable choice. Unrealistic in comparison to something one would do professionally, it was also limited in terms of performance and scalability. I wanted to take something built for school and adapt it to a real-world environment. This meant rethinking the architecture, improving the data flow, and enhancing the overall user experience to meet modern web standards.
